# ONE world

!!! note "Status — In Progress"
    <span style="color:#ec4899;font-weight:700;">●</span> ONE world is in active development. The project page at [world.expl.one](https://world.expl.one) marks its place in the ecosystem; the in-world experience is being built.

## What it does and what's coming

ONE world is intended as the metaverse layer of the ONE ecosystem — a place where digital experiences across the ecosystem can connect. The work today is focused on building that environment; specific features and scope may evolve as design decisions are made.

### Today you can…

- Visit the project's home at [world.expl.one](https://world.expl.one) to see where ONE world fits inside the ecosystem.
- Follow progress through the broader ecosystem channels.

### In progress…

- A virtual environment layer where users can spend time inside ONE world.
- Digital identity integration so accounts can carry across ONE projects.
- An in-world economy aligned with the rest of the ONE ecosystem.

Specific delivery dates are not committed here; broader timing lives in the roadmap, and scope may be refined as the project develops.

## How it fits

ONE world is one of the user-facing surfaces planned for the ONE ecosystem and shares its foundation with the other ONE projects.

- It is intended to lean on the same shared backbone as the rest of the projects — see [ONE network](../one-network/introduction.md).
- Identity is intended to flow in from the ecosystem-wide identity layer — see [ONE id](../one-id/introduction.md).
- It is one of several projects coordinated under the [Projects Overview](../projects/overview.md).
- Like every ONE project, it is operated through nodes — see the [EXPL Nodes introduction](../nodes/introduction.md).
- Return to the [documentation home](../index.md) to see how the projects connect.

## Current state

ONE world is marked **In Progress**. The user-facing experience is being designed and built rather than running today, and the scope listed above should be read as direction rather than commitment. Features, capabilities, and timelines may change before the first public release.
